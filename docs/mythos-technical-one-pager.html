Mythos / Aperture

A real-time, dual-agent system design interview simulator built on Next.js and Convex. The product combines a formal interviewer lane, a specialist teammate lane, shared session memory, live progress signals, and an evidence-backed report with PDF export.

System Responsibility Split

Convex is the canonical source of truth for session state, messages, notes, counters, signals, and report artifacts. LLMs are used only for visible dialogue, structured turn analysis, and final report JSON. Next.js owns the interactive room, report UI, OCR ingress, and the server-side PDF route.

End-to-End Runtime

  1. Session creation: `createSession` builds an interview blueprint, inserts Convex records, seeds scratch notes and workspace content, then creates a dedicated interviewer thread plus teammate thread.
  2. Live interview: the UI subscribes to room state, messages, streaming partials, notes, and status via Convex queries; candidate sends land in `channels.sendCandidateMessage`.
  3. AI response cycle: `orchestratorNode.processCandidateTurn` loads cross-channel context, calls the active agent, streams partial output when enabled, persists the visible reply, then runs hidden turn analysis.
  4. Session completion: `endSession` transitions to `generating_report`, `reportingNode.generateReport` creates structured report JSON, and the report page exposes on-demand PDF export through Playwright.

Core Data Model

  • Interview entities: `sessions`, `channels`, `messages`, `events`.
  • High-churn runtime state: `agentRuntimeState`, `streamingResponses`, `signalState`.
  • Shared memory: `sessionState`, `sessionFacts`, `phaseSummaries`.
  • Candidate-private work: `scratchNotes`, `solutionWorkspaces`.
  • Evaluation outputs: `annotations`, `metricSnapshots`, `reports`, `reportArtifacts`.

Key Technical Decisions

Realtime
Convex subscriptions drive live tabs, timers, notes, and streaming reply state.
Isolation
Each visible channel has its own agent thread, but both read the same condensed session context.
Safety
Message send rate limiting, explicit public/internal function boundaries, and validator-backed APIs.
Scalability
High-write operational state is split into dedicated tables instead of large session blobs.
Flexibility
LLM provider can switch between Gemini and OpenRouter without changing orchestration code.

Current Scope

  • Hackathon-style MVP with public session IDs instead of full authentication.
  • System design interview focus with standardized phases, nudges, stress, and teammate collaboration.
  • Report PDF is rendered on demand today; schema hooks already exist for persisted PDF artifacts.
